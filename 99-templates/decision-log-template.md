## Guidelines

- When to add a decision: changes to data model or migrations, API/Kafka contract changes, security/privacy trade-offs, rollback strategy, or non-obvious architectural choices.
- What NOT to log: trivial implementation details, small refactors, or purely cosmetic changes (use code comments instead).
